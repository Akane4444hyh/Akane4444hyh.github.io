Dying Man Last Delusion

Nobody knows what a customized desktop means to a thirsty Narcissus.

It’s not just about easily reading official Docs or writing simple Scripts,

It is a countless debate with poor free LLM, endless searching for solutions for my NVIDIA 5060, and hysterically staring at dazzling screen for 15 hours until 3 a.m. just for adjusting a little button to a right place and play its own role.

A Long Journey

Before Arch, I had some basic Linux experience.

  • Tried Ubuntu in VMware
  • Used Linux Mint for daily learning
  • Created bootable USB with Ventoy

Everything worked well.

It felt like a perfect start.

But things changed after I moved to vanilla Arch Linux

Installing Vanilla Arch Linux

I installed Arch Linux using Ventoy and the official ISO.

Basic Setup

a.Connect to the internet using iwd

b.Enable time synchronization using timedatectl

timedatectl set-ntp true

c.Configure mirrors using reflector and archlinux-keyring

d.Create partitions using GPT and cfdisk :

lsblk -f

find the prepared disk space

Partition Type Size
EFI ESP 512M
Swap Swap 16G
Root Btrfs Remaining

e. Formatting

mkfs.fat -F32 /dev/nvme0n1p1
mkswap /dev/nvme0n1p3
swapon /dev/nvme0n1p3
mkfs.btrfs /dev/nvme0n1p2

f. Btrfs Subvolumes
Create subvolumes(mount temporarily)

mount /dev/nvme0n1p2 /mnt 
btrfs subvolume create /mnt/@ 
btrfs subvolume create /mnt/@home 
umount /mnt

Mount subvolumes and EFI

mount -t btrfs -o subvol=@,compress=zstd /dev/nvme0n1p2 /mnt 
mkdir /mnt/home 
mount -t btrfs -o subvol=@home,compress=zstd /dev/nvme0n1p2 /mnt/home 
mkdir -p /mnt/efi 
mount /dev/nvme0n1p1 /mnt/efi

Install Base System

a.Install necessary

pacstrap -K /mnt base base-devel linux-zen linux-firmware btrfs-progs vim networkmanager sudo intel-ucode

b.Generate fstab

genfstab -U /mnt > /mnt/etc/fstab

c.CHROOT!!!!!

arch-chroot /mnt

Enable network service:

systemctl enable NetworkManager

c1.Configure timezone using timedatectl 

ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
hwclock --systohc

c2.Localize

vim /etc/locale.gen

uncomment en_US UTF-8

locale-gen

vim /etc/locale.conf

write:”LANG=en_US.UTF-8”

c3.Hostname

vim /etc/hostname

write: hostname-you-like

c4.Root
set root passwd

passwd

c5.Install grub and efibootmgr

pacman -S grub efibootmgr

(Since customized the EFI mount point)

grub-install --target=x86_64-efi --efi-directory=/efi --bootloader-id=name-you-like

grub-mkconfig -o /boot/grub/grub.cfg

c6.If dual-boot System

pacman -S os-prober exfat-utils

edit /etc/default/grub
uncomment the line has “os-prober”

grub-mkconfig -o /boot/grub/grub.cfg

REBOOT!!!!

change boot options

–

the first problem

Issue: black screen after enter grub

Cause: nouveau not supporting NVIDIA 5060

Solution: add “nouveau.modeset=0 nvidia-drm.modeset=1” to “GRUB_CMDLINE_LINUX_DEFAULT” in /etc/default/grub

–

Initial Config

pacman -Syu

a.Add regular user

useradd -mG wheel username-you-like
passwd username-you-like

visudo

uncomment %wheel All=…

b.Install snapper

pacman -S snapper snap-pac grub-btrfs inotify-tools
systemctl enable --now grub-btrfs

REBOOT!!!!
config

snapper -c root create-config /
snapper -c home create-config /home

grub-mkconfig -o /boot/grub/grub.cfg
